# Feature Index - Quick Reference

## New Features Since v1 — Quick Navigation

### 1. **Treg TCR-Contact Self-Similarity Scoring (Criterion 4)**
- **File:** `src/scoring/scorer.py` (lines 307-355)
- **What:** Replaces broken IL-10 model with evidence-based TCR-contact analysis
- **Key Function:** `score_treg_tcr_contact(peptide, predictions_df) → float`
- **Data:** 2.3M unique TCR-facing motifs from human proteome
- **Result:** 0.27-0.999 range for ITP (good discrimination)
- **See Full Details:** `docs/NEW_FEATURES_v2.md` Section 1

---

### 2. **IL-10 Data Leakage Investigation**
- **File:** `docs/esm2_upgrade_log.md` 
- **What:** Discovered IL-10 model (AUC 0.91 on full data) → AUC 0.50 on non-overlapping subset
- **Root Cause:** Only 394 positive examples; insufficient for generalization
- **Decision:** IL-10 weight set to 0.00 (kept as placeholder)
- **Impact:** Validates TCR-contact biology over black-box IL-10 prediction
- **See Full Details:** `docs/NEW_FEATURES_v2.md` Section 2

---

### 3. **ESM-2 Embedding Infrastructure**
- **File:** `src/scoring/esm_embeddings.py` (147 lines)
- **What:** 320-dim protein language model embeddings with caching
- **Model:** `esm2_t6_8M_UR50D` (smallest ESM-2)
- **Status:** Available but ESM-2 didn't improve IL-10 model
- **Future Use:** Better IL-10 training datasets
- **See Full Details:** `docs/NEW_FEATURES_v2.md` Section 3

---

### 4. **Cathepsin S Processing Scoring**
- **File:** `src/scoring/processing.py` (132 lines)
- **What:** Predicts likelihood peptide generated by cathepsin S (dominant MHC-II protease)
- **Algorithm:** P2 (3x weight) + P1 + P1' log-odds at cleavage sites
- **Status:** Integrated, experimental (not in composite score yet)
- **Literature:** Choe 2006, Biniossek 2011, MEROPS database
- **See Full Details:** `docs/NEW_FEATURES_v2.md` Section 4

---

### 5. **Population Coverage (Bui Formula)**
- **File:** `src/assembly/population_coverage.py` (189 lines)
- **What:** Fraction of population that can present ≥1 epitope
- **Formula:** Coverage = 1 - ∏(1 - p_phenotype_i) with Hardy-Weinberg
- **Coverage:** 4 populations (European, East_Asian, African, South_Asian)
- **Alleles:** 12 total (9 DRB1, 2 DQ, 1 DP)
- **See Full Details:** `docs/NEW_FEATURES_v2.md` Section 5

---

### 6. **Disease Profile System (Generalization)**
- **File:** `src/data/disease_profile.py` (107 lines)
- **What:** JSON-based disease configuration (ITP, MS, extensible to any autoimmune disease)
- **Required Fields:** disease_id, primary_antigen, gold_standard_path, calibration spec
- **Profiles:** `data/diseases/itp.json`, `data/diseases/ms.json`
- **Key Function:** `load_disease_profile(disease_id) → dict`
- **Impact:** No code changes to add new disease
- **See Full Details:** `docs/NEW_FEATURES_v2.md` Section 6

### 9. **Scoring CLI with Disease Support**
- **Entry Point:** `python -m src.scoring.scorer --disease itp`
- **Location:** `src/scoring/scorer.py` (lines 877-1031)
- **Features:**
  - Disease profile loading
  - Antigen sequence validation (UniProt + position check)
  - Predictions enrichment (core_peptide from TSVs)
  - Gold standard validation

  - Calibration check

- **Output Files:**

## QUICK START

### Run ITP Pipeline
```bash
python -m src.scoring.scorer --disease itp
```

### Run MS Pipeline
```bash
python -m src.scoring.scorer --disease ms
```

### Add New Disease
1. Create `data/diseases/{disease_id}.json` with profile
2. Create `data/processed/{disease_id}_gold_standard.json` with validated peptides
3. Run: `python -m src.scoring.scorer --disease {disease_id}`
